## 上下文

当前应用所有 UI 文本都是硬编码的中文字符串。项目使用 React 19 + TypeScript + Zustand 技术栈。需要引入 i18n 国际化支持，实现中英文双语切换。

## 目标 / 非目标

**目标：**
- 引入 react-i18next 国际化框架
- 创建中英文翻译文件
- 替换所有硬编码文本为翻译键
- 添加语言切换 UI
- 保存用户语言偏好

**非目标：**
- 支持动态语言加载
- 支持 RTL 布局
- 翻译第三方库内容

## 决策

1. **选择 react-i18next 而非 react-intl**
   - react-i18next 生态更成熟，文档更完善
   - 与 React 集成更好，支持 hooks
   - 社区活跃度高

2. **翻译文件结构**
   - 按功能模块组织（common, chat, settings 等）
   - 支持嵌套键名
   - 中英文文件分离

3. **语言状态管理**
   - 使用 Zustand store 管理当前语言
   - 持久化到 electron-store

4. **语言切换 UI**
   - 在设置对话框中添加语言选项
   - 支持即时切换，无需重启

## 风险 / 权衡

- **风险**: 翻译遗漏 → 缓解: 使用 fallback 机制，默认显示中文
- **风险**: 性能影响 → 缓解: 翻译文件预加载，避免运行时加载
- **权衡**: 增加包体积 → 可接受，翻译文件体积小
