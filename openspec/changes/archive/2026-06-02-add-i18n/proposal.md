## 为什么

当前应用所有 UI 文本都是硬编码的中文字符串，无法支持多语言。随着产品面向更广泛的用户群体，需要支持中英文双语切换，提升用户体验和产品国际化能力。

## 变更内容

- 引入 i18n 国际化框架（react-i18next）
- 创建中英文翻译文件
- 替换所有组件中的硬编码文本为翻译键
- 添加语言切换功能
- 保存用户语言偏好设置

## 功能 (Capabilities)

### 新增功能
- `i18n`: 国际化基础设施，包括翻译文件管理、语言切换、文本翻译功能

### 修改功能
- `chat-ui`: 替换聊天界面中的硬编码文本（输入框、按钮等）
- `session-persistence`: 保存用户语言偏好设置

## 影响

- **前端组件**: 所有包含硬编码文本的 React 组件需要修改
- **状态管理**: 需要添加语言状态到 Zustand store
- **依赖**: 新增 react-i18next、i18next 依赖
- **文件结构**: 新增 locales/ 目录存放翻译文件
- **用户体验**: 用户可在设置中切换中英文
