## 新增需求

### 需求： 系统必须支持中英文双语
系统必须支持中文和英文两种语言，用户可以在设置中切换语言。

#### 场景： 切换语言
- **当** 用户在设置中选择"English"
- **那么** 系统界面文本立即切换为英文

#### 场景： 语言偏好持久化
- **当** 用户切换语言后重启应用
- **那么** 系统保持用户上次选择的语言

### 需求： 翻译文件必须按模块组织
翻译文件必须按功能模块组织，支持嵌套键名。

#### 场景： 翻译文件结构
- **当** 开发者查看翻译文件
- **那么** 文件按 common、chat、settings 等模块组织

### 需求： 系统必须提供翻译函数
系统必须提供 React hooks 和组件方式的翻译函数。

#### 场景： 使用翻译 hook
- **当** 组件需要翻译文本
- **那么** 可以使用 useTranslation hook 获取翻译函数

#### 场景： 翻译键缺失
- **当** 翻译键不存在
- **那么** 系统显示键名作为 fallback
