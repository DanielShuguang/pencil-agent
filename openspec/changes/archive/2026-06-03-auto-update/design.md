## Context

electron-updater 已经在 package.json 中作为依赖，但尚未集成。需要实现完整的自动更新流程。

**当前状态：**
- electron-updater 已安装
- electron-builder 已配置
- 没有更新相关的代码

**约束：**
- 需要支持 GitHub Releases 或自定义更新服务器
- 需要处理网络错误和更新失败情况
- 需要支持 Windows 和 macOS

## Goals / Non-Goals

**Goals:**
- 实现自动检查更新（应用启动时）
- 实现手动检查更新（用户触发）
- 显示更新进度和状态
- 支持更新通知
- 支持跳过版本

**Non-Goals:**
- 不实现增量更新
- 不实现回滚机制
- 不实现自定义更新服务器

## Decisions

### 1. 更新策略：自动检查 + 手动触发

**决策**：应用启动时自动检查更新，用户也可以手动检查

**理由**：
- 自动检查确保用户及时获得更新
- 手动触发给予用户控制权
- 平衡用户体验和更新及时性

### 2. 更新流程：后台下载 + 提示安装

**决策**：发现更新后后台下载，下载完成后提示用户安装

**理由**：
- 不打断用户当前工作
- 下载完成后给予用户选择权
- 支持稍后安装

### 3. 更新服务器：GitHub Releases

**决策**：使用 GitHub Releases 作为更新服务器

**理由**：
- 免费、可靠
- 与代码仓库集成
- 支持版本管理和发布说明

## Risks / Trade-offs

**[风险] 网络问题导致更新失败** → 实现重试机制和错误处理

**[风险] 更新过程中应用崩溃** → 支持断点续传

**[权衡] 自动下载 vs 手动下载** → 选择自动下载，但给予用户安装选择权

## Migration Plan

1. **Phase 1：更新服务**

3. **Phase 3：UI 组件**
