## Why

当前应用只有暗色主题，用户在明亮环境下使用时屏幕反光严重，影响可读性和使用体验。实现暗色/亮色主题切换可以：
- 提升不同环境下的可读性
- 减少长时间使用的眼睛疲劳
- 跟随系统主题偏好，提供一致的用户体验

**扩展性要求**：架构设计需要支持后续添加更多主题（如护眼模式、高对比度模式等），而非硬编码两种主题。

## What Changes

- 实现可扩展的主题系统：基于设计令牌和 CSS 变量的主题架构
- 添加主题切换 UI：在设置或标题栏添加切换按钮
- 支持系统主题跟随：使用 `nativeTheme` API 自动检测系统偏好
- 持久化主题选择：保存用户选择到本地存储
- 更新所有组件样式：确保所有 UI 组件支持主题切换

## Capabilities

### New Capabilities
- `theme-system`: 可扩展主题系统核心架构，包括主题注册机制、设计令牌定义、CSS 变量生成、主题切换逻辑
- `theme-persistence`: 主题偏好的持久化存储和系统主题跟随

### Modified Capabilities

## Impact

- **前端组件**：所有 UI 组件需要更新为使用 CSS 变量或 Tailwind 主题令牌
- **状态管理**：需要新增 theme-store 管理主题状态
- **主进程**：需要使用 Electron 的 `nativeTheme` API 检测系统主题
- **构建配置**：Tailwind 配置需要支持主题切换
- **类型定义**：需要定义主题相关的 TypeScript 类型
