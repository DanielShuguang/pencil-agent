## 为什么

当前 Agent 的工具执行缺乏权限控制。Agent 可直接执行 bash 命令、读写任意文件，无需用户确认。Docker 不可用时代码在宿主机直接执行，无沙箱隔离。这带来安全风险：误删文件、执行危险命令、访问敏感数据。随着 Agent 能力增强，必须建立细粒度的权限控制系统，让用户对 Agent 行为有知情权和控制权。

## 变更内容

- 新增工具执行前的用户确认机制（可配置自动/手动模式）
- 新增文件路径访问控制（工作目录白名单、敏感路径黑名单）
- 新增危险命令拦截（`rm -rf`、`sudo`、`chmod 777` 等）
- 新增工具级别权限配置（按工具粒度启用/禁用）
- 新增执行日志审计（持久化工具调用记录）
- 修改现有工具执行流程，注入权限检查层

## 功能 (Capabilities)

### 新增功能

- `tool-permission`: 工具执行权限控制核心，包括确认机制、路径限制、命令拦截
- `execution-audit`: 工具执行日志审计，持久化记录所有工具调用及结果
