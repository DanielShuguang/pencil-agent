## ADDED Requirements

### 需求：工具执行日志记录
系统必须记录所有工具调用的完整信息，包括输入参数和执行结果。

#### 场景：记录成功执行
- **当** 任意工具执行完成且成功
- **那么** 系统必须记录：时间戳、会话 ID、工具名称、输入参数、执行结果、耗时

#### 场景：记录失败执行
- **当** 任意工具执行完成且失败
- **那么** 系统必须记录：时间戳、会话 ID、工具名称、输入参数、错误信息、耗时

#### 场景：记录被拒绝的执行
- **当** 工具执行被权限系统拒绝
- **那么** 系统必须记录：时间戳、会话 ID、工具名称、输入参数、拒绝原因

### 需求：日志持久化存储
执行日志必须持久化存储在本地，支持按会话查询。

#### 场景：日志写入本地存储
- **当** 工具执行产生日志
- **那么** 日志必须异步写入本地存储（JSON Lines 格式）
- **且** 不得阻塞工具执行流程

#### 场景：按会话查询日志
- **当** 用户请求查看某会话的执行日志
- **那么** 系统必须返回该会话下所有工具调用记录
- **且** 记录按时间正序排列

### 需求：日志查看界面
系统必须提供日志查看界面，让用户了解 Agent 的工具调用历史。

#### 场景：查看当前会话日志
- **当** 用户在聊天界面点击"查看执行日志"
- **那么** 系统必须显示当前会话的工具调用列表
- **且** 每条记录显示工具名称、参数摘要、状态（成功/失败/拒绝）、耗时

#### 场景：查看日志详情
- **当** 用户点击某条日志记录
- **那么** 系统必须展开显示完整参数和结果

### 需求：日志清理
系统必须支持清理过期日志，防止存储无限增长。

#### 场景：自动清理过期日志
- **当** 日志存在超过 30 天
- **那么** 系统必须在应用启动时自动清理过期日志

#### 场景：手动清理日志
- **当** 用户在设置中点击"清理日志"
- **那么** 系统必须删除所有历史日志
- **且** 删除前必须弹出确认框
