## ADDED Requirements

### 需求：工具执行确认模式
系统必须支持三种工具执行确认模式：`auto`（自动执行）、`prompt`（每次确认）、`smart`（智能判断）。用户必须能在设置中切换模式。

#### 场景：auto 模式下直接执行
- **当** 确认模式设置为 `auto`
- **且** Agent 调用任意工具
- **那么** 工具必须直接执行，不弹出确认框

#### 场景：prompt 模式下弹出确认
- **当** 确认模式设置为 `prompt`
- **且** Agent 调用任意工具
- **那么** 系统必须弹出确认框，显示工具名称、参数
- **且** 用户点击"允许"后才执行
- **且** 用户点击"拒绝"后返回错误给 Agent

#### 场景：smart 模式下智能判断
- **当** 确认模式设置为 `smart`
- **且** Agent 调用低风险工具（read、grep、find、ls）
- **那么** 工具必须直接执行
- **当** Agent 调用高风险工具（write、edit、bash）
- **那么** 系统必须弹出确认框

### 需求：危险命令拦截
系统必须拦截已知的危险 shell 命令，在执行前要求用户确认。拦截规则必须可配置。

#### 场景：拦截 rm -rf 命令
- **当** Agent 调用 bash 工具
- **且** 命令包含 `rm -rf` 或 `rm -r` 且目标路径非 `/tmp` 下
- **那么** 系统必须弹出高风险警告确认框
- **且** 显示将要删除的路径

#### 场景：拦截 sudo 命令
- **当** Agent 调用 bash 工具
- **且** 命令包含 `sudo`
- **那么** 系统必须拒绝执行并返回错误信息

#### 场景：拦截权限修改命令
- **当** Agent 调用 bash 工具
- **且** 命令包含 `chmod 777` 或 `chmod -R 777`
- **那么** 系统必须弹出高风险警告确认框

### 需求：文件路径访问控制
系统必须限制 Agent 可访问的文件路径范围。

#### 场景：允许访问工作目录内文件
- **当** Agent 调用 read/write/edit 工具
- **且** 目标路径在当前工作目录内
- **那么** 工具必须正常执行

#### 场景：拒绝访问工作目录外文件
- **当** Agent 调用 read/write/edit 工具
- **且** 目标路径不在当前工作目录内
- **且** 目标路径不在用户配置的额外允许路径列表中
- **那么** 系统必须拒绝执行并返回权限错误

#### 场景：拒绝访问敏感路径
- **当** Agent 调用 read/write/edit 工具
- **且** 目标路径匹配敏感路径规则（如 `~/.ssh`、`~/.gnupg`、系统目录）
- **那么** 系统必须拒绝执行，无论是否在工作目录内

### 需求：工具级别权限配置
系统必须允许用户按工具粒度配置启用/禁用。

#### 场景：禁用特定工具
- **当** 用户在设置中禁用 `bash` 工具
- **且** Agent 调用 `bash` 工具
- **那么** 系统必须返回工具已禁用的错误

#### 场景：启用所有默认工具
- **当** 用户未修改工具权限配置
- **那么** 系统必须默认启用 `read`、`write`、`edit`、`bash` 四个工具

### 需求：权限确认弹窗
系统必须提供清晰的权限确认弹窗，展示足够的信息供用户决策。

#### 场景：弹窗显示工具信息
- **当** 需要用户确认工具执行
- **那么** 弹窗必须显示：工具名称、工具参数、风险等级
- **且** 必须提供"允许"和"拒绝"按钮
- **且** 必须提供"本次会话记住选择"选项

#### 场景：会话级记住选择
- **当** 用户勾选"本次会话记住选择"并点击"允许"
- **且** 同一会话内再次调用相同工具
- **那么** 系统必须直接执行，不再弹出确认框
