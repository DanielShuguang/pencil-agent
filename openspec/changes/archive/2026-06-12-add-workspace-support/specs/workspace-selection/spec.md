## 新增需求

### 需求：创建会话时选择工作空间目录

用户创建新会话时，系统必须弹出操作系统原生目录选择对话框，用户选择目录后才能完成会话创建。

#### 场景：成功选择目录创建会话
- **当** 用户点击"新建会话"按钮
- **那么** 系统弹出操作系统原生目录选择对话框
- **当** 用户选择一个有效目录并确认
- **那么** 系统创建会话，将该目录路径作为 `cwd` 绑定到会话

#### 场景：取消目录选择
- **当** 用户在目录选择对话框中点击"取消"
- **那么** 系统不创建会话，返回到会话列表

#### 场景：选择不存在或无权限的目录
- **当** 用户选择一个不存在或无读取权限的目录
- **那么** 系统必须显示错误提示，不创建会话

### 需求：工作空间路径持久化

会话的工作空间路径（`cwd`）必须持久化到本地存储，应用重启后可恢复。

#### 场景：重启后恢复会话工作空间
- **当** 用户关闭并重新打开应用
- **那么** 会话列表中每个会话仍显示其绑定的工作空间
- **当** 用户点击一个历史会话
- **那么** 系统使用持久化的 `cwd` 恢复 Agent 会话

#### 场景：旧会话数据兼容
- **当** 应用加载一个没有 `cwd` 字段的旧会话记录
- **那么** 系统必须将该会话标记为"需要重新选择工作空间"
- **当** 用户点击该会话
- **那么** 系统提示用户重新选择目录

### 需求：工作空间路径传播到 Agent 子系统

会话的 `cwd` 必须传播到所有文件操作相关子系统。

#### 场景：ResourceLoader 使用会话级 cwd
- **当** Agent 会话创建时
- **那么** `DefaultResourceLoader` 必须使用会话级 `cwd` 而非 `process.cwd()`

#### 场景：权限检查使用会话级 cwd
- **当** Agent 执行文件读写工具时
- **那么** 权限系统必须使用会话级 `cwd` 解析相对路径

#### 场景：子进程沙箱使用会话级 cwd
- **当** Agent 在沙箱中执行 bash 命令时
- **那么** 子进程的 `spawn` 必须传入会话级 `cwd` 作为工作目录

### 需求：IPC 目录选择接口

系统必须提供 IPC 接口供渲染进程调用目录选择对话框。

#### 场景：渲染进程调用目录选择
- **当** 渲染进程调用 `dialog:selectDirectory`
- **那么** 主进程调用 `dialog.showOpenDialog` 并返回选择结果
- **当** 用户选择目录
- **那么** 返回 `{ canceled: false, filePaths: ["/selected/path"] }`

#### 场景：主窗口引用获取
- **当** 目录选择对话框被触发
- **那么** 系统必须以当前聚焦窗口为父窗口打开模态对话框
