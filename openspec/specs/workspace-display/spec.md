## 新增需求

### 需求：侧边栏会话项显示项目名

侧边栏每个会话项必须显示其工作空间的项目名（路径最后一级目录名）。

#### 场景：显示项目名
- **当** 会话列表渲染一个会话项
- **那么** 该项必须显示 📁 图标和路径最后一级目录名
- **当** 工作空间路径为 `/Users/dev/projects/frontend`
- **那么** 显示 `📁 frontend`

#### 场景：项目名溢出处理
- **当** 项目名超出侧边栏宽度
- **那么** 文本必须截断并显示省略号
- **当** 用户悬停在项目名上
- **那么** 必须显示完整路径的 tooltip

### 需求：StatusBar 显示完整工作空间路径

StatusBar 必须显示当前会话的完整工作空间路径。

#### 场景：StatusBar 展示路径
- **当** 用户进入一个有工作空间的会话
- **那么** StatusBar 左侧必须显示 📁 图标和完整路径

#### 场景：StatusBar 无会话状态
- **当** 没有活跃会话
- **那么** StatusBar 不显示工作空间信息

### 需求：路径不存在提示

当用户打开一个工作空间目录已被删除或移动的会话时，系统必须给出明确提示。

#### 场景：工作空间目录不存在
- **当** 用户点击一个历史会话
- **且** 该会话绑定的工作空间目录不存在或无权限访问
- **那么** 系统必须显示 toast 错误提示"工作空间不存在：{路径}"
- **且** 不进入该会话
